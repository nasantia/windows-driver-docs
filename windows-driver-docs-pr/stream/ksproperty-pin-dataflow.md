# KSPROPERTY\_PIN\_DATAFLOW


The KSPROPERTY\_PIN\_DATAFLOW property specifies the direction of data flow on pins instantiated by the pin factory. Sink pins are entry points into a filter; source pins output from a filter.

Remarks
-------

Specify the pin factory in the **PinId** member of the [**KSP\_PIN**](https://docs.microsoft.com/windows-hardware/drivers/ddi/ks/ns-ks-ksp_pin) structure.

KSPROPERTY\_PIN\_DATAFLOW returns an enumeration of type [**KSPIN\_DATAFLOW**](https://docs.microsoft.com/windows-hardware/drivers/ddi/ks/ne-ks-kspin_dataflow), set to either **KSPIN\_DATAFLOW\_IN** or KSPIN\_DATAFLOW\_OUT.

Stream minidrivers do not need to handle this property directly; the stream class driver handles this property using stream request blocks to query for more information.
